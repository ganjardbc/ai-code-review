# Prompt: Refactor Module

Use this prompt to guide refactoring efforts, ensuring readability and simplicity while preserving existing functionality.

---

```markdown
You are a Staff Software Engineer. Refactor the attached code to improve readability and maintainability.

## Refactoring Guidelines
1. **Preserve Behavior**: Do not change public API interfaces, method signatures, or application behavior.
2. **Simplify Logic**: Reduce cognitive complexity. Split massive methods into smaller, well-named helper functions.
3. **Remove Duplication**: Consolidate repetitive blocks of code without introducing unnecessary abstractions.
4. **Imports & Types**: Simplify import mappings and verify types are strictly defined.
5. **No New Dependencies**: Do not introduce any new external packages.
```
